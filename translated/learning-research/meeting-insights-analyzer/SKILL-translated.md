# 会议洞察分析器

此技能将您的会议记录转换为关于您沟通模式的可操作洞察，帮助您成为一个更有效的沟通者和领导者。

## 何时使用此技能

- 分析跨多个会议的沟通模式
- 获取关于您的领导力和主持风格的反馈
- 识别您何时避免艰难对话
- 了解您的说话习惯和填充词
- 跟踪沟通技能随时间的变化
- 准备带有具体例子的绩效评估
- 指导团队成员的沟通风格

## 此技能的功能

1. **模式识别**：识别会议中反复出现的行为，例如：
   - 避免冲突或间接沟通
   - 说话比例和轮流
   - 提问与陈述模式
   - 积极倾听指标
   - 决策方法

2. **沟通分析**：评估沟通的有效性：
   - 清晰度和直接性
   - 填充词和含糊语言的使用
   - 语调和情感模式
   - 会议控制和主持

3. **可操作反馈**：提供具体、带时间戳的示例，包括：
   - 发生了什么
   - 为什么这很重要
   - 如何改进

4. **趋势跟踪**：在分析多个会议时比较模式

## 如何使用

### 基本设置

1. 将您的会议记录下载到一个文件夹中（例如，`~/meetings/`)
2. 在 Claude Code 中导航到该文件夹
3. 请求您想要的分析

## 指令

当用户请求会议分析时：

1. **发现可用数据**
   - 扫描文件夹以查找记录文件（.txt、.md、.vtt、.srt、.docx）
   - 检查文件是否包含演讲者标签和时间戳
   - 确认会议的日期范围
   - 在记录中识别用户的姓名/标识符

2. **明确分析目标**

   如果未指定，询问他们想了解什么：
   - 具体的行为（避免冲突、打断、填充词）
   - 沟通的有效性（清晰度、直接性、倾听）
   - 会议主持技能
   - 说话模式和比例
   - 改进的增长领域

3. **分析模式**

   对于每个请求的洞察：

   **避免冲突**：
   - 寻找含糊语言（“也许”、“有点”、“我认为”）
   - 间接表达而不是直接请求
   - 在出现紧张时改变话题
   - 不承诺同意（“是的，但是...”）
   - 不解决明显的问题

   **说话比例**：
   - 计算会议中花费在说话上的百分比
   - 计算打断次数（由用户发起和接收）
   - 测量平均说话轮次长度
   - 跟踪问题与陈述的比例

   **填充词**：
   - 计算“嗯”、“啊”、“像”、“你知道”、“实际上”等填充词的数量
   - 注意每分钟或每说话轮次的发生频率
   - 识别它们增加的情况（紧张、不确定）

   **积极倾听**：
   - 引用他人先前观点的问题
   - 复述或总结他人的想法
   - 建立在他人贡献的基础上
   - 提出澄清问题

   **领导力与主持**：
   - 决策方法（指令性 vs. 协作性）
   - 如何处理分歧
   - 包含较安静参与者
   - 时间管理和议程控制
   - 后续和行动项的清晰度

4. **提供具体示例**

   对于每个发现的模式，包括：

   ```markdown
   ### [模式名称]

   **发现**：[关于该模式的简短总结]

   **频率**：[X次在Y次会议中]

   **示例**：

   1. **[会议名称/日期]** - [时间戳]

      **发生了什么**：
      > [记录中的实际引语]

      **为什么这很重要**：
      [关于影响或错失机会的解释]

      **更好的方法**：
      [具体的替代措辞或行为]

   [重复 2-3 个最强的例子]
   ```

5. **综合洞察**

   在分析所有模式后，提供：

   ```markdown
   # 会议洞察摘要

   **分析周期**：[日期范围]
   **分析的会议**：[X次会议]
   **总时长**：[X小时]

   ## 识别出的关键模式

   ### 1. [主要模式]
   - **观察到的**：[你所看到的]
   - **影响**：[为什么这很重要]
   - **建议**：[如何改进]

   ### 2. [第二模式]
   [相同的结构]

   ## 沟通优势

   1. [优势 1 及示例]
   2. [优势 2 及示例]
   3. [优势 3 及示例]

   ## 改进的增长机会

   1. **[领域 1]**：[具体、可操作的忠告]
   2. **[领域 2]**：[具体、可操作的忠告]
   3. **[领域 3]**：[具体、可操作的忠告]

   ## 说话统计数据

   - 平均说话时间：[X%的会议]
   - 提出的问题：[每次会议平均X个]
   - 填充词：[每分钟X个]
   - 打断：[每次会议给出/接收Y个]

   ## 下一步行动

   [3-5 个具体的改进沟通的行动]
   ```

6. **提供后续选项**
   - 在未来的会议中跟踪这些相同的指标
   - 深入研究特定的会议或模式
   - 与行业标准进行比较
   - 制定个人沟通发展计划
   - 为绩效评估生成摘要

```markdown
# 冲突避免模式

在 15 次会议中发现了 23 个您使用间接沟通或避免直接解决紧张的时刻。

## 模式：对关键反馈的含糊其辞

**频率**：在 7 次会议中的 8 次

**示例 1：与 Sarah 的 1:1 会议** - 00:14:32

**为什么这很重要**：
您需要解决 Sarah 的项目落后于进度的问题，但含糊语言（“也许”、“可能”、“如果你认为”）和结尾时的推诿（“ whatever 你认为”）使她很容易忽略紧迫性。

**更好的方法**：
“Sarah，项目落后了两个星期。我们需要讨论阻碍进展的原因，并今天制定一个新的时间表。”

## 建议

1. **直接命名问题**在第一句话中
2. **移除含糊词汇**如“也许”、“有点”、“sort of”
3. **提出具体问题**而不是暗示
4. **安排艰难的对话**而不是随意提出
```

## 设置技巧

### 获取会议记录

**从 Granola**（免费，需订阅 Lenny 的通讯）：
- Granola 自动转录您的会议
- 将转录的记录导出到一个文件夹：[如何操作的说明]
- 将 Claude Code 指向该文件夹

**从 Zoom**：
- 启用带有转录的云录制
- 会议结束后下载 VTT 或 SRT 文件
- 将其存储在专用文件夹中

**从 Google Meet**：
- 使用 Google Docs 自动转录
- 将转录的文档保存到文件夹中
- 将其作为 .txt 文件下载或允许 Claude Code 访问

**从 Fireflies.ai、Otter.ai 等**：
- 批量导出转录
- 将其存储在本地文件夹中
- 在文件夹上运行分析

### 最佳实践

1. **一致的命名**：使用 `YYYY-MM-DD - 会议名称.txt` 格式
2. **定期分析**：每月或每季度审查一次以查看趋势
3. **具体查询**：每次询问一种行为以获得深度
4. **隐私**：将敏感会议数据保留在本地
5. **以行动为导向**：一次专注于一个改进领域
