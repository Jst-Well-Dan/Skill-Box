# 六顶思考帽框架

## 概述

一种从六个不同角度审视想法的结构化方法。一次戴一顶帽子，以避免混淆。

**目的：** 分离思考模式，减少冲突，确保全面分析。

## 六顶帽子

### 🎩 白帽：事实与数据

**焦点：** 仅客观信息

**问题：**
- 我们有什么事实？
- 可用的数据是什么？
- 缺少什么信息？
- 数字是什么？
- 我们如何验证这一点？

**语言：**
- "根据……"
- "数据显示……"
- "我们知道……"
- "我们不知道……"

**在内容分析中：**
- 提取所有事实主张
- 记录引用来源
- 识别缺失数据
- 将事实与解释分开

**模板：**
```markdown
## 已知事实
- [可验证的事实 1]
- [可验证的事实 2]

## 缺失信息
- [ ] [差距 1]
- [ ] [差距 2]

## 数据来源
- [引用 1]: [质量评级]
```

---

### 🎩 红帽：情感与直觉

**焦点：** 肠胃感觉，直觉，直观反应

**问题：**
- 我的即时反应是什么？
- 我的直觉怎么说？
- 这激发了我什么情感？
- 我的直觉告诉我什么？

**语言：**
- "我感觉……"
- "我的直觉说……"
- "这让我不舒服，因为……"
- "我对此很兴奋……"

**在内容分析中：**
- 在阅读时注意情感反应
- 识别触发强烈情感的因素
- 信任关于可信度的直觉
- 认识到文本中的说服性情感诉求

**模板：**
```markdown
## 即时反应
- [情感] 在 [具体部分]

## 肠胃感觉
- 感觉不对劲的是：[直觉]
- 令人兴奋/有说服力的是：[引起共鸣的内容]

## 文本中的情感触发器
- 作者使用 [情感] 来说服
```

**重要提示：** 无需证明。这是纯粹的感觉。

---

### 🎩 黑帽：谨慎与风险

**焦点：** 批判性判断，潜在问题

**问题：**
- 什么可能会出错？
- 有什么风险？
- 什么是不现实的？
- 弱点在哪里？
- 有什么缺点？

**语言：**
- "问题是……"
- "这不会奏效，因为……"
- "这里的风险是……"
- "我们忽视了……"

**在内容分析中：**
- 识别逻辑缺陷
- 记录未经证实的说法
- 指出潜在失败
- 询问可行性
- 考虑意外后果

**模板：**
```markdown
## 逻辑弱点
- [推理中的缺陷 1]

## 应用风险
- 风险：[后果]
- 当它失败时：[情景]

## 缺失考虑
- 没有考虑到：[因素]

## 过度乐观的说法
- [说法] 似乎不切实际，因为 [原因]
```

**重要提示：** 这是批判性的，而不是愤世嫉俗的。这是风险评估所必需的。

---

### 🎩 黄帽：利益与乐观

**焦点：** 正面方面，价值，机会

**问题：**
- 这里的价值是什么？
- 有什么好处？
- 为什么这会奏效？
- 最佳情况是什么？
- 这创造了什么机会？

**语言：**
- "好处是……"
- "这可能奏效，因为……"
- "这里的价值是……"
- "一个机会是……"

**在内容分析中：**
- 提取有价值的见解
- 识别强有力的论点
- 记录实际应用
- 发现新颖的视角
- 认识到推动领域发展的内容

**模板：**
```markdown
## 关键价值主张
- 有价值的见解：[有用的内容]

## 强有力的论点
- 有力支持：[论点]

## 潜在应用
- 可用于：[用例]

## 新颖贡献
- 对：[主题] 的新视角

## 最佳情况结果
如果完全应用：[积极情景]
```

**重要提示：** 要现实但慷慨。找到真正的价值。

---

### 🎩 绿帽：创造性与替代方案

**焦点：** 新想法，可能性，创新

**问题：**
- 还有什么是可能的？
- 我们如何以另一种方式思考这个问题？
- 有什么创造性的替代方案？
- 如果我们将 X 与 Y 结合会怎样？
- 什么是不寻常的？

**语言：**
- "如果……"
- "另一种看待它的方式……"
- "我们也可以……"
- "一个替代方案是……"

**在内容分析中：**
- 将作者的想法进一步扩展
- 生成对提出的解决方案的替代方案
- 将其他框架结合起来
- 应用到新的领域
- 创造性地挑战假设

**模板：**
```markdown
## 想法的扩展
- 将 [概念] 进一步发展：[新应用]

## 替代方法
而不是 [作者的方法]，如果：[替代方案]

## 创造性组合
- [文本中的想法] + [其他框架] = [新见解]

## 非传统应用
- 将此应用于 [意外的领域]

## "如果"情景
- 如果 [假设] 被反转，会发生什么？
```

**重要提示：** 绿帽模式下不进行批评。所有想法都受欢迎。

---

### 🎩 蓝帽：过程与元思考

**焦点：** 管理思考，概述，结论

**问题：**
- 我们涵盖了什么？
- 我们需要哪种思考模式？
- 摘要是什么？
- 下一步是什么？
- 我们应该如何思考这个问题？

**语言：**
- "我们已经涵盖了……"
- "下一步是……"
- "总结一下……"
- "我们需要关注……"

**在内容分析中：**
- 决定何时使用哪些帽子
- 综合所有帽子的发现
- 确定下一步
- 制定学习策略
- 监控分析过程

**模板：**
```markdown
## 分析过程
1. 白：收集事实
2. 红：注意反应  
3. 黑：识别风险
4. 黄：找到价值
5. 绿：生成替代方案

## 综合分析
结合所有视角：
- [综合见解]

## 下一步
- [ ] 进一步研究：[领域]
- [ ] 实践测试：[行动]
- [ ] 深入研究：[主题]

## 时间投入
值得 [X] 时间，因为 [原因]
```

**重要提示：** 蓝帽组织其他帽子。它是指挥者。

## 使用模式

### 快速评估（15 分钟）
白 → 红 → 黄 → 黑 → 蓝

### 深度分析（60 分钟）  
白 → 红 → 黑 → 黄 → 绿 → 蓝 → （重新访问任何帽子）→ 蓝

### 问题解决
白 → 红 → 绿 → 黄 → 黑 → 绿 → 蓝

### 决策制定
白 → 黄 → 黑 → 红 → 蓝

## 参与规则

1. **一次一顶帽子** - 不要混合模式
2. **每个人都戴同一顶帽子** - 并行思考（如果团体）
3. **将人与帽子分开** - 你不是 "批判性的人"，你是在戴黑帽
4. **时间限制** - 为每顶帽子设定限制
5. **蓝帽控制** - 决定顺序和时机

## 常见错误

❌ **混合帽子** - "这很好（黄）但风险很大（黑）"
✅ **分开：** 黄色会议："这很好，因为 X。" 然后 黑色会议："风险是 Y。"

❌ **评判感觉** - "那是不理性的" 在红帽期间
✅ **接受感觉**：所有情感都被记录下来，没有评判

❌ **黄帽薄弱** - "我想这里有一些价值……"
✅ **真正的乐观**：真正找到好处

❌ **停留时间过长** - 每顶帽子 30 分钟
✅ **继续前进**：每顶帽子通常 5-10 分钟就足够了

## 阅读中的应用

**何时使用哪顶帽子：**

- **理解阶段：** 白（事实），红（反应）
- **评估阶段：** 黑（批评），黄（价值）
- **应用阶段：** 绿（想法），蓝（综合）

**完整周期：**
1. 白：文本实际上说了什么？
2. 红：我对它有什么感觉？
3. 黑：它有什么问题？
4. 黄：它有什么好处？
5. 绿：我们可以用这个做什么？
6. 蓝：所以呢？接下来是什么？

## 与其他模型的整合

- **在第一性原理之后：** 使用帽子来评估重建的论点
- **在系统映射之后：** 使用黄/黑对杠杆点
- **在批判性思维之前：** 使用红来揭示偏见
- **在综合期间：** 使用蓝来组织发现
