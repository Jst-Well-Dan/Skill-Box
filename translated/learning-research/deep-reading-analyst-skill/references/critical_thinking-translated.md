# 批判性思维框架

## 论证质量评估

### 证据评估矩阵

| 证据类型 | 强度 | 警示信号 |
|----------|----------|-----------|
| 同行评审研究 | 高 | 样本量，利益冲突 |
| 原始数据 | 高 | 收集方法，偏见 |
| 专家共识 | 中-高 | 领域共识 vs. 单一专家 |
| 案例研究 | 中 | 选择性偏见，可推广性 |
| 轶事 | 低 | 不具代表性 |
| “研究表明...”（无引用）| 非常低 | 模糊，不可验证 |

### 逻辑谬误清单

**因果谬误：**
- [ ] 后果先因（A在B之前 ≠ A导致B）
- [ ] 相关性 ≠ 因果性
- [ ] 过于简化的原因（单一因素解释复杂现象）

**证据谬误：**
- [ ] 选择性证据（选择性证据）
- [ ] 存活者偏差（只有成功的案例可见）
- [ ] 仓促归纳（小样本 → 广泛主张）

**修辞谬误：**
- [ ] 诉诸人身（攻击人，而非论点）
- [ ] 诉诸权威（没有相关领域的专业知识）
- [ ] 摆设论点（曲解对手立场）
- [ ] 滑坡谬误（没有正当理由的极端结果）
- [ ] 假的二分法（只有两种选择，而实际上更多）

**统计谬误：**
- [ ] 忽略基线概率
- [ ] 绝对风险 vs. 相对风险混淆
- [ ] 谬导的平均数（均值隐藏分布）

## 批判性提问协议

### 第1级：理解
- 核心主张是什么？
- 有什么证据支持它？
- 关键假设是什么？

### 第2级：分析
- 证据是否足以支持主张？
- 有逻辑漏洞吗？
- 这个论点有什么缺失？

### 第3级：评估
- 这个论点整体有多强？
- 什么可以加强/削弱它？
- 有哪些替代解释？

### 第4级：综合
- 这与其他知识如何相符？
- 尽管有缺陷，作者可能在哪些方面是正确的？
- 有什么慈善的解释？

## 源头可信度评估

**作者背景：**
- 该领域的相关专业知识？
- 可能的利益冲突？
- 准确性的记录？

**出版背景：**
- 同行评审？编辑标准？
- 原始来源还是解释？
- 出版日期（时效性）？

**动机分析：**
- 作者的目标是什么？（提供信息/说服/销售）
- 谁从这一主张中受益？
- 目标受众是什么？

## 反证搜索

在分析强烈主张时，积极寻找：
1. 有相反发现的研究
2. 专家意见分歧
3. 失败的复制
4. 边界条件（何时不起作用？）

## 平衡评估模板

```markdown
## 论证优势
- [什么是得到良好支持的]
- [强有力的证据指向]

## 论证劣势  
- [逻辑漏洞]
- [薄弱或缺失的证据]

## 未解答的问题
- [论点没有涉及的内容]

## 条件真理
这个论点在以下情况下最强：[上下文]
这个论点在以下情况下最弱：[上下文]
```

## 钢铁人练习

在批评之前，构建**最强可能的论点**：
1. 仁慈地填补逻辑漏洞
2. 添加最佳可能的支持证据
3. 解决明显的反对意见
4. 然后评估这个最强版本

这可以防止攻击摆设论点，并确保公平评估。
