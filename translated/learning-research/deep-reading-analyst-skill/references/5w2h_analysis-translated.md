# 5W2H 分析框架

## 概述

5W2H 是一种系统性的提问方法，通过 7 个基本问题全面理解任何主题、问题或计划。

**7 个核心问题**：
- **What** - 对象、目标、内容
- **Why** - 原因、目的、重要性
- **Who** - 人员、利益相关者、责任
- **When** - 时间、日程、截止日期
- **Where** - 地点、背景、范围
- **How** - 方法、流程、步骤
- **How much** - 成本、数量、指标

## 何时使用

适用于：
- 📋 快速理解完整文章信息
- 🔍 发现信息缺口和遗漏
- 📊 评估计划可行性
- 💼 分析商业案例和提案
- 📝 组织复杂信息
- ✅ 验证信息完整性

## 7 个问题的详细说明

### W1: What（内容）

**核心问题**：
1. 主要主题是什么？
2. 关键主张/结论是什么？
3. 提出了哪些解决方案？
4. 涉及哪些核心概念？
5. 预期结果是什么？

**深入探究**：
- 真正的问题是什么？
- 缺少了什么？
- 可能出错的地方是什么？
- 有哪些替代方案？

### W2: Why（原因）

**核心问题**：
1. 为什么现在讨论这个主题？
2. 为什么这个解决方案有效？
3. 为什么不选择其他方法？
4. 为什么受众应该关心？
5. 为什么这个时间点？

**五问法技术**：
问“为什么”5次以找到根本原因：
```
问题：[表面问题]
→ 为什么 1：[第一层]
  → 为什么 2：[第二层]
    → 为什么 3：[第三层]
      → 为什么 4：[第四层]
        → 为什么 5：[根本原因] ← 真正的问题
```

### W3: Who（人员）

**核心问题**：
1. 目标受众是谁？
2. 利益相关者是谁？
3. 谁负责执行？
4. 谁受益？谁受损？
5. 谁的专业知识/案例被引用？
6. 作者是谁？（潜在的偏见？）

**深入分析**：
- 利益相关者映射
- 决策者 vs. 执行者
- 谁有否决权？

### W4: When（时间）

**核心问题**：
1. 何时开始？
2. 时间表/日程是什么？
3. 结果何时出现？
4. 这有多紧急？
5. 何时需要关键决策？
6. 历史背景是什么？

**时间陷阱识别**：
- 不切实际的时间期望？
- 缺少关键里程碑？
- 考虑了缓冲时间？

### W5: Where（背景）

**核心问题**：
1. 这适用于哪里？（地理、行业、组织）
2. 执行在哪里发生？
3. 哪里可以获取资源？
4. 限制/约束是什么？
5. 影响范围是什么？

**背景依赖性**：
- 在不同的环境中是否适用？
- 文化差异？
- 规模考虑？

### H1: How（方法）

**核心问题**：
1. 具体步骤是什么？
2. 使用哪些工具/方法？
3. 如何衡量进度？
4. 如何处理障碍？
5. 如何确保质量？
6. 如何开始？

**流程图**：
```
[开始] → [步骤 1] → [决策点]
                         ↓ 是 / ↓ 否
                    [步骤 2A]  [步骤 2B]
                         ↓         ↓
                    [步骤 3] ← [汇聚]
                         ↓
                    [结束]
```

### H2: How much（指标）

**核心问题**：
1. 成本是多少？（金钱、时间、机会、学习）
2. 需要多少资源？
3. 预期回报率是多少？
4. 规模/幅度是多少？
5. 风险有多大？
6. 目标指标是什么？

**回报率分析**：
```
投资：
- 直接成本：[$]
- 间接成本：[$]
- 总计：[$]

回报：
- 预期收益：[$]
- 时间框架：[X 个月]
- 回报率 = (收益 - 成本) / 成本 × 100%

值得吗？[是/否]
```

## 实际示例

### 示例：远程工作文章

**文章标题**： "如何成功过渡到远程工作"

```markdown
## 5W2H 分析

### What
- 主题：公司从办公室到远程工作的过渡
- 解决方案：分阶段过渡计划
- 目标：保持或提高团队生产力

### Why
- 背景：疫情后的工作模式变化
- 价值：节省成本，提高灵活性
- 理由：成熟的技术 + 工作性质的变化
⚠️ 缺失：为什么不考虑混合模式？

### Who
- 目标：小型企业管理者
- 执行者：人力资源 + IT 团队
- 受益者：员工（灵活性）+ 公司（成本）
⚠️ 缺失：如何处理抵制者？

### When
- 推荐：3 个月过渡
  - 第 1 个月：试点
  - 第 2 个月：扩展
  - 第 3 个月：全面推出
✅ 清晰

### Where
- 适用：知识型工作
- 不适用：制造业、服务业
- 工具：Zoom、Slack、Asana
✅ 清晰界限

### How
1. 建立远程工作政策
2. 配置技术基础设施
3. 培训经理和员工
4. 建立虚拟协作规范
5. 定期评估和调整
✅ 清晰步骤

### How Much
- 工具成本：$50/人/月
- 培训：40 小时
- 回报率：办公室成本下降 50%
⚠️ 缺失：过渡期生产力下降成本

## 差距分析

缺失关键信息：
1. Why - 为什么不考虑混合模式
2. Who - 如何处理抵制者
3. How much - 过渡期隐性成本

风险：
- 中等风险：80% 信息完整
- 需要补充：变革管理和隐性成本

建议：
1. 研究混合工作模式
2. 制定变革沟通计划
3. 预算 15-20% 用于过渡期生产力下降
```

## 高级技巧

### 1. 优先级 5W2H

不是所有 7 个问题都同等重要——根据文章类型进行调整：

```markdown
策略文章：Why（40%）> What（30%）> How（20%）> 其他
如何文章：How（40%）> What（25%）> How much（20%）> 其他
分析文章：What（30%）> Why（30%）> Who（20%）> 其他
```

### 2. 迭代 5W2H

```markdown
第一轮：快速扫描所有 7 个问题（15 分钟）
第二轮：深入分析 2-3 个最缺乏的部分（30 分钟）
第三轮：交叉验证答案一致性（15 分钟）
```

### 3. 扩展 5W2H (+ 2H)

添加两个额外的 How：
- **How to measure?**（指标）
- **How to sustain?**（维护）

## 常见陷阱

### ❌ 表面分析
- 错误：机械地回答 7 个问题而不深入
- 正确：深入挖掘每个问题的 2-3 个层次

### ❌ 忽略差距
- 错误：只分析文章提供的内容
- 正确：明确指出文章没有回答的内容

### ❌ 隔离思维
- 错误：独立分析 7 个问题
- 正确：寻找答案之间的一致性和矛盾

## 实践技巧

1. **打印清单**：准备标准的 5W2H 模板
2. **计时练习**：在 15 分钟内完成快速分析
3. **反向验证**：使用分析重建文章——是否完整？
4. **比较应用**：将 5W2H 应用于同一主题的多个文章

## 总结

5W2H 是：
- ✅ **完整性检查器**：确保没有信息遗漏
- ✅ **快速理解框架**：系统地捕捉本质
- ✅ **沟通桥梁**：标准化的分析语言

最佳用于：
- 📋 项目计划评估
- 📊 商业提案分析
- 📝 操作指南理解
- 🔍 信息差距识别

**记住**：5W2H 的价值不仅在于回答这 7 个问题，还在于发现哪些问题 *没有* 回答——以及为什么。
