# 完成开发分支

## 概述

通过提供清晰的选择和处理选定的工作流程来指导开发工作的完成。

**核心原则：** 验证测试 → 提供选项 → 执行选择 → 清理。

**开始时宣布：** "我正在使用完成开发分支技能来完成这项工作。"

## 流程

### 第 1 步：验证测试

**在提供选项之前，验证测试是否通过：**

```bash
# 运行项目的测试套件
npm test / cargo test / pytest / go test ./...
```

**如果测试失败：**
```
测试失败（<N> 个失败）。在完成合并/PR 之前必须修复：

[显示失败]

测试未通过，无法进行合并/PR。
```

停止。不要进行到第 2 步。

**如果测试通过：** 继续到第 2 步。

### 第 2 步：确定基础分支

```bash
# 尝试常见的基分支
git merge-base HEAD main 2>/dev/null || git merge-base HEAD master 2>/dev/null
```

或者询问："这个分支是从 main 分支分叉出来的 - 这正确吗？"

### 第 3 步：提供选项

提供以下 exactly 4 个选项：

```
实现完成。您想做什么？

1. 本地合并回 <base-branch>
2. 推送并创建一个 Pull Request
3. 保持分支不变（我稍后处理）
4. 放弃这项工作

选择哪个选项？
```

**不要添加解释** - 保持选项简洁。

### 第 4 步：执行选择

#### 选项 1：本地合并

```bash
# 切换到基分支
git checkout <base-branch>

# 拉取最新版本
git pull

# 合并功能分支
git merge <feature-branch>

# 验证合并结果的测试
<test command>

# 如果测试通过
git branch -d <feature-branch>
```

然后：清理工作树（第 5 步）

#### 选项 2：推送并创建 PR

```bash
# 推送分支
git push -u origin <feature-branch>

# 创建 PR
gh pr create --title "<title>" --body "$(cat <<'EOF'
## 摘要
<2-3 个关于更改的要点>

## 测试计划
- [ ] <验证步骤>
EOF
)"
```

然后：清理工作树（第 5 步）

#### 选项 3：保持不变

报告："保持分支 <name>。工作树保存在 <path>。"

**不要清理工作树。**

#### 选项 4：放弃

**首先确认：**
```
这将永久删除：
- 分支 <name>
- 所有提交：<commit-list>
- 工作树在 <path>

输入 'discard' 以确认。
```

等待确切的确认。

如果确认：
```bash
git checkout <base-branch>
git branch -D <feature-branch>
```

然后：清理工作树（第 5 步）

### 第 5 步：清理工作树

**对于选项 1、2、4：**

检查是否在工作树中：
```bash
git worktree list | grep $(git branch --show-current)
```

如果是：
```bash
git worktree remove <worktree-path>
```

**对于选项 3：** 保持工作树。

## 快速参考

| 选项 | 合并 | 推送 | 保持工作树 | 清理分支 |
|--------|-------|------|---------------|----------------|
| 1. 本地合并 | ✓ | - | - | ✓ |
| 2. 创建 PR | - | ✓ | ✓ | - |
| 3. 保持不变 | - | - | ✓ | - |
| 4. 放弃 | - | - | - | ✓ (强制) |

## 常见错误

**跳过测试验证**
- **问题：** 合并损坏的代码，创建失败的 PR
- **修复：** 在提供选项之前始终验证测试

**开放式问题**
- **问题：** "我接下来该做什么？" → 不明确
- **修复：** 提供正好 4 个结构化选项

**自动工作树清理**
- **问题：** 在可能需要它的时候删除工作树（选项 2、3）
- **修复：** 只为选项 1 和 4 清理

**没有放弃的确认**
- **问题：** 错误地删除工作
- **修复：** 需要输入 "discard" 确认

## 红色警报

**永远不要：**
- 在测试失败的情况下进行操作
- 在不验证结果测试的情况下合并
- 在没有确认的情况下删除工作
- 在没有明确请求的情况下强制推送

**始终：**
- 在提供选项之前验证测试
- 提供正好 4 个选项
- 对选项 4 进行输入确认
- 只为选项 1 和 4 清理工作树
