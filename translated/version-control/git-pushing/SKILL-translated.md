# Git 推送工作流程

暂存所有变更，创建一个传统的提交信息，并推送到远程分支。

## 使用场景

当用户：
- 明确要求推送变更（“推送这个”、“提交并推送”）
- 提及保存工作到远程（“保存到 GitHub”、“推送到远程”）
- 完成了一个功能并想要分享
- 说类似“让我们把这个推上去”或“提交这些变更”的话时，会自动激活。

## 工作流程

**始终使用脚本** - 不要使用手动 git 命令：

```bash
bash skills/git-pushing/scripts/smart_commit.sh
```

使用自定义信息：

```bash
bash skills/git-pushing/scripts/smart_commit.sh "feat: 添加功能"
```

脚本处理：暂存、传统的提交信息、Claude 页脚、带有 -u 标志的推送。
