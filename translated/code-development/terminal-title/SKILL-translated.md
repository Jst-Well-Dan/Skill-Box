# 终端标题

## 概述

根据 Claude 正在执行的任务自动设置描述性的终端窗口标题。对于运行多个 Claude Code 实例的开发者来说，这对于快速识别哪个终端正在处理哪个任务至关重要。

## 使用场景

**始终触发此技能：**
- 在每个新的 Claude Code 会话的开始（在收到第一个用户提示后）
- 当切换到明显不同的任务时（例如，从“API 集成”切换到“数据库迁移”）

**在以下任务切换时触发：**
- 从前端工作切换到后端工作
- 从调试切换到新功能开发
- 从一个模块/组件切换到完全不同的另一个
- 开始系统不同部分的工作（例如，从身份验证切换到支付）

**不触发的情况：**
- 关于同一任务的后续问题（“你能给那个函数添加注释吗？”）
- 对当前工作的微小改进（“把它变成蓝色而不是红色”）
- 调试你刚刚构建的功能
- 说明（“你说的 X 是什么意思？”）
- 对同一组件或模块进行迭代
- 中途任务状态更新或进度检查

## 工作原理

1. **提取任务摘要**：分析用户的提示以识别高级任务
2. **生成标题**：创建简洁、描述性的标题（最多 40 个字符）
3. **设置标题**：使用生成的标题执行 `scripts/set_title.sh` 脚本
4. **无需确认**：这将在后台自动完成

## 标题格式指南

**好的标题：**
- "API 集成：身份验证流程"
- "修复：登录错误"
- "DB 迁移：用户表"
- "构建：仪表板 UI"
- "重构：支付模块"

**不好的标题：**
- 太长："实现具有 OAuth2.0 支持的新身份验证系统"（超过 40 个字符）
- 太泛："工作"或"编码"
- 太冗长："用户想要我帮助他们做……"

**格式模式：**
```
[动作/类别]：[具体焦点]
```

保持标题简洁、可操作和易于识别。

## 避免的常见错误

**❌ 太冗长：**
- 差："正在实现具有 JWT 令牌的用户身份验证系统"
- 好："构建：JWT 身份验证"

**❌ 太模糊：**
- 差："编码一些东西"
- 差："工作"
- 好："重构：API 层"

**❌ 包含系统信息：**
- 差："john-macbook-pro：调试应用"
- 差："/Users/john/project：构建功能"
- 好："调试：应用问题"

**❌ 使用完整的句子：**
- 差："我正在构建仪表板组件"
- 好："构建：仪表板 UI"

## 实现

**执行标题脚本：**
```bash
bash scripts/set_title.sh "你的标题"
```

**示例工作流程：**
```bash
# 用户询问："帮助我调试 API 中的身份验证流程"
bash scripts/set_title.sh "调试：Auth API 流程"

# 用户询问："创建用户个人资料页面的 React 组件"
bash scripts/set_title.sh "构建：用户个人资料 UI"

# 用户询问："为支付处理模块编写测试"
bash scripts/set_title.sh "测试：支付模块"
```

## 脚本详情

`scripts/set_title.sh` 脚本使用 ANSI 转义序列来设置终端标题。它与以下终端兼容：
- macOS 终端
- iTerm2
- Alacritty
- 大多数现代终端模拟器（xterm、rxvt、screen、tmux）

脚本接受一个参数（标题字符串），如果没有提供标题则退出静默（安全行为）。

## 可选自定义

用户可以通过设置 `CLAUDE_TITLE_PREFIX` 环境变量来可选地自定义终端标题的前缀：

```bash
export CLAUDE_TITLE_PREFIX="🤖 Claude"
```

这将生成如下标题：`🤖 Claude | 构建：仪表板 UI`

没有前缀，标题保持标准格式：`构建：仪表板 UI`

**注意：**您不需要检查此变量或修改您的行为。脚本会自动处理。
