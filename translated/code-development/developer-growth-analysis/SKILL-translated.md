# 开发者成长分析

此技能通过分析您的 Claude Code 聊天互动，对您最近的编码工作提供个性化反馈，并识别揭示优势和成长领域的模式。

## 何时使用此技能

当您想：
- 了解您最近工作的开发模式和习惯
- 识别具体的技术差距或反复出现的问题
- 发现哪些主题需要更深入的学习
- 获取针对您实际工作模式量身定制的学习资源
- 跟踪您最近项目中的改进领域
- 找到直接针对您正在培养的技能的高质量文章

此技能非常适合希望在没有等待代码审查的情况下获得结构化反馈的开发者，以及希望从自己的工作历史中获得数据驱动洞察的开发者。

## 此技能的功能

此技能对您的开发工作执行六步分析：

1. **读取您的聊天历史**：访问过去 24-48 小时的本地 Claude Code 聊天历史，了解您一直在做什么。

2. **识别开发模式**：分析您正在解决的问题类型、使用的技术、遇到的问题以及您如何处理不同类型的工作。

3. **检测改进领域**：识别表明技能差距、反复出现的挑战、低效的方法或可能从更深入的知识中受益的领域的模式。

4. **生成个性化报告**：创建一个全面的报告，显示您的工作摘要、识别的改进领域和具体增长建议。

5. **查找学习资源**：使用 HackerNews 精选与您的改进领域直接相关的高质量文章和讨论，为您提供针对实际开发工作的阅读列表。

6. **发送到您的 Slack 私信**：自动将完整的报告发送到您自己的 Slack 私信，以便您随时随地进行参考。

## 如何使用

要求 Claude 分析您最近的编码工作：

```
分析我最近的聊天中的开发者成长
```

或者更具体地说明您要分析的哪个时间段：

该技能将生成一个格式化的报告，包括：
- 您最近工作的概述
- 识别的关键改进领域
- 每个领域的具体建议
- 来自 HackerNews 的精选学习资源
- 您可以专注的行动项

## 指令

当用户请求分析其开发者成长或编码模式时：

1. **访问聊天历史**

   从 `~/.claude/history.jsonl` 读取聊天历史。此文件是 JSONL 格式，其中每行包含：
   - `display`：用户的消息/请求
   - `project`：正在工作的项目
   - `timestamp`：Unix 时间戳（以毫秒为单位）
   - `pastedContents`：任何粘贴的代码或内容

   根据当前时间戳筛选过去 24-48 小时的条目。

2. **分析工作模式**

   从筛选的聊天中提取和分析以下内容：
   - **项目和领域**：用户正在做什么类型的项目？（例如，后端、前端、DevOps、数据等）
   - **使用的技术**：在对话中出现了哪些语言、框架和工具？
   - **问题类型**：正在解决哪些类别的问题？（例如，性能优化、调试、功能实现、重构、设置/配置）
   - **遇到的问题**：用户遇到了哪些问题？寻找：
     - 关于类似主题的重复问题
     - 需要多次尝试才能解决的问题
     - 表明知识差距的问题
     - 复杂的架构决策
   - **方法模式**：用户如何解决问题？（例如，有条理的、探索性的、实验性的）

3. **识别改进领域**

   根据分析，识别 3-5 个用户可以改进的具体领域。这些应该是：
   - **具体的**（不是像“提高编码技能”这样的模糊描述）
   - **基于证据的**（基于实际的聊天历史）
   - **可操作的**（可以做出的实际改进）
   - **优先排序的**（最有影响力的优先）

   优秀的改进领域示例：
   - “高级 TypeScript 模式（泛型、实用类型、类型守卫）- 您在 [特定项目] 中在类型安全性方面遇到了困难”
   - “错误处理和验证 - 我注意到您修补了几个与缺失空检查相关的错误”
   - “Async/await 模式 - 您最近的工作显示了一些竞争条件和时序问题”
   - “数据库查询优化 - 您多次重写了相同的查询”

4. **生成报告**

   创建一个具有以下结构的全面报告：

   ```markdown
   # 您的开发者成长报告

   **报告周期**：[昨天 / 今天 / [自定义日期范围]]
   **最后更新**：[当前日期和时间]

   ## 工作摘要

   [2-3 段总结用户做了什么，涉及的项目，使用的技术，以及整体重点领域]

   示例：
   "在过去 24 小时内，您主要专注于后端开发，涉及三个不同的项目。您的工作涉及 TypeScript、React 和部署基础设施。您处理了各种挑战，包括功能实现、调试和架构决策，特别关注 API 设计和数据库优化。"

   ## 改进领域（优先排序）

   ### 1. [领域名称]

   **为什么这很重要**：[解释为什么这项技能对用户的工作很重要]

   **我观察到什么**：[来自聊天历史的特定证据，表明这个差距]

   **建议**：[在这个领域改进的具体步骤]

   **提升技能所需时间**：[对所需努力的简要估计]

   ---

   [重复 2-4 个其他领域]

   ## 观察到的优势

   [2-3 个突出显示您做得好的要点 - 要继续做的事情]

   ## 行动项

   优先顺序：
   1. [从最高优先级改进领域派生的行动项]
   2. [下一个领域的行动项]
   3. [下一个领域的行动项]

   ## 学习资源

   [将在下一步中填充]
   ```

5. **搜索学习资源**

   使用 Rube MCP 搜索 HackerNews 与每个改进领域相关的文章：

   - 对于每个改进领域，构建针对高质量资源的搜索查询
   - 使用 RUBE_SEARCH_TOOLS 通过以下查询在 HackerNews 中搜索：
     - “学习 [技术/模式] 最佳实践”
     - “[技术] 高级模式和技巧”
     - “在 [语言] 中调试 [特定问题类型]”

     - 文章标题
     - 发布日期
     - 简要说明为什么它与您的改进领域相关
     - 文章链接

   将此部分添加到报告中：

6. **呈现完整的报告**

   以干净、可读的格式交付报告，用户可以：
   - 快速扫描关键要点
   - 用于专注的学习计划
   - 在专注于改进的一周后参考
   - 如果他们想要外部反馈，可以与导师分享

7. **将报告发送到 Slack 私信**

   使用 Rube MCP 将完整的报告发送到用户的 Slack 私信：

   - 通过 RUBE_SEARCH_TOOLS 检查 Slack 连接是否处于活动状态

     - 将报告标题和周期作为第一条消息发送
     - 将报告分成逻辑部分（摘要、改进、优势、行动、资源）
     - 将每个部分格式化为结构良好的 Slack 消息，并使用正确的 markdown

   - 在 CLI 输出中确认交付

   这确保用户可以在他们经常检查的地方获得报告，并在整个星期内参考它。

### 1. 高级 TypeScript 模式和类型安全性

**为什么这很重要**：TypeScript 对您的工作至关重要，但利用其高级功能（泛型、实用类型、条件类型、类型守卫）可以显着提高代码可靠性并减少运行时错误。更好的类型安全性在编译时而不是在生产中捕获错误。

**我观察到什么**：在您最近的聊天中，您正在处理连接数据结构，并在正确地配置身份验证配置方面遇到了几次困难。您还必须对不同的连接状态迭代联合类型。有机会更有效地使用区分联合和类型守卫。

**建议**：学习 TypeScript 的高级类型系统，特别是实用类型（Omit、Pick、Record）、条件类型和区分联合。将这些模式应用到您的连接配置处理和身份验证状态管理中。

**提升技能所需时间**：5-8 小时的专注学习和实践

### 2. 安全数据处理和 UI 中的信息隐藏

**为什么这很重要**：您识别并修复了一个安全问题，其中敏感的连接数据被显示在您的控制台中。防止信息泄露对于处理用户凭据和 API 密钥的应用程序至关重要。良好的做法可以防止安全事件和用户信任违规。

**我观察到什么**：您注意到“您的应用程序”页面显示了完整的连接数据，包括身份验证配置。这表明您有良好的安全直觉，下一步是将此纳入您处理敏感信息的默认思维。

**建议**：审查处理前端应用程序中敏感数据的安全最佳实践。创建在显示到 UI 之前过滤/屏蔽敏感信息的可重用模式。考虑实现一个明确白名单可以显示在 UI 中的安全数据层。

**提升技能所需时间**：3-4 小时

### 3. 组件架构和响应式 UI 模式

**为什么这很重要**：您正在设计需要跨不同屏幕尺寸和用户交互工作的 UI。强大的组件架构可以更容易地构建没有错误的复杂 UI，并提高可维护性。

**建议**：学习 React 组件组合模式和 CSS 布局最佳实践（特别是弹性框和网格）。关注防止溢出问题的容器查询和响应式模式。了解组件组合库和设计系统方法。
