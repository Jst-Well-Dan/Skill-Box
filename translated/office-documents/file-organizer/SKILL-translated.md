# 文件整理器

这项技能充当您的个人组织助理，帮助您在计算机上维护一个干净、逻辑的文件结构，而无需不断手动组织带来的心理负担。

## 何时使用此技能

- 您的下载文件夹一团糟
- 您找不到文件，因为它们散落在各个地方
- 您有占用空间的重复文件
- 您的文件夹结构不再有意义
- 您想建立更好的组织习惯
- 您正在开始一个新项目，需要良好的结构
- 您在归档旧项目之前正在清理

## 此技能的功能

1. **分析当前结构**：审查您的文件夹和文件，了解您有什么
2. **查找重复文件**：识别系统中的重复文件
3. **建议组织**：根据您的文件内容提出合理的文件夹结构
4. **自动化清理**：在您的批准下移动、重命名和组织文件
5. **维护上下文**：根据文件类型、日期和内容做出明智的决定
6. **减少杂乱**：识别您可能不再需要的旧文件

## 指令

当用户请求文件组织帮助时：

1. **了解范围**

   提出澄清问题：
   - 哪个目录需要组织？（下载、文档、整个家目录？）
   - 主要问题是什么？（找不到东西、重复、太乱、没有结构？）
   - 有任何文件或文件夹要避免？（当前项目、敏感数据？）
   - 采取多积极的组织方式？（保守型 vs. 全面清理？）

2. **分析当前状态**

   审查目标目录：
   ```bash
   # 获取当前结构的概览
   ls -la [目标目录]
   
   # 检查文件类型和大小
   find [目标目录] -type f -exec file {} \; | head -20
   
   # 识别最大的文件
   du -sh [目标目录]/* | sort -rh | head -20
   
   # 统计文件类型
   find [目标目录] -type f | sed 's/.*\.//' | sort | uniq -c | sort -rn
   ```
   
   总结发现：
   - 文件和文件夹总数
   - 文件类型细分
   - 大小分布
   - 日期范围
   - 明显的组织问题

3. **识别组织模式**

   根据文件，确定逻辑分组：
   
   **按类型**：
   - 文档（PDFs、DOCX、TXT）
   - 图片（JPG、PNG、SVG）
   - 视频（MP4、MOV）
   - 归档（ZIP、TAR、DMG）
   - 代码/项目（包含代码的目录）
   - 电子表格（XLSX、CSV）
   - 演示文稿（PPTX、KEY）
   
   **按目的**：
   - 工作 vs. 个人
   - 活动 vs. 归档
   - 项目特定
   - 参考材料
   - 临时/草稿文件
   
   **按日期**：
   - 当前年/月
   - 前几年
   - 非常旧（归档候选）

4. **查找重复文件**

   当请求时，搜索重复文件：
   ```bash
   # 通过哈希查找精确重复文件
   find [目录] -type f -exec md5 {} \; | sort | uniq -d
   
   # 查找具有相同名称的文件
   find [目录] -type f -printf '%f\n' | sort | uniq -d
   
   # 查找大小相似的文件
   find [目录] -type f -printf '%s %p\n' | sort -n
   ```
   
   对于每一组重复文件：
   - 显示所有文件路径
   - 显示大小和修改日期
   - 建议保留哪个（通常是最新或命名最好的）
   - **重要**：在删除之前始终请求确认

5. **提出组织计划**

   在做出更改之前提出明确的计划：

   ```markdown
   # [目录]的组织计划
   
   ## 当前状态
   
   - X个文件分布在Y个文件夹中
   - [大小] 总计
   - 文件类型：[细分]
   - 问题：[列出问题]
   
   ## 建议的结构
   
   ```
   [目录]/
   ├── 工作/
   │   ├── 项目/
   │   ├── 文档/
   │   └── 归档/
   ├── 个人/
   │   ├── 照片/
   │   ├── 文档/
   │   └── 媒体/
   └── 下载/
       ├── 待整理/
       └── 归档/
   ```
   
   ## 我将做出的更改
   
   1. **创建新文件夹**：[列表]
   2. **移动文件**：
      - X个PDF文件 → 工作/文档/
      - Y个图片 → 个人/照片/
      - Z个旧文件 → 归档/
   3. **重命名文件**：[任何重命名模式]
   4. **删除**：[重复文件或垃圾文件]
   
   ## 需要您决定的文件
   
   - [列出您不确定的任何文件]
   
   准备好继续吗？（是/否/修改）

6. **执行组织**

   在获得批准后，系统地组织：

   ```bash
   # 创建文件夹结构
   mkdir -p "新文件夹路径"
   
   # 明确记录地移动文件
   mv "旧路径/文件.pdf" "新路径/文件.pdf"
   
   # 使用一致的模式重命名文件
   # 示例： "YYYY-MM-DD - 描述.ext"
   ```

   **重要规则**：
   - 在删除任何内容之前始终确认
   - 记录所有移动以备撤销
   - 保留原始修改日期
   - 优雅地处理文件名冲突
   - 在遇到意外情况时停止并请求确认

7. **提供总结和维护技巧**

   在组织后：

   ```markdown
   # 组织完成！✨
   
   ## 变化
   
   - 创建了[X]个新文件夹
   - 组织了[Y]个文件
   - 通过删除重复文件释放了[Z] GB
   - 归档了[W]个旧文件
   
   ## 新结构
   
   [显示新的文件夹树]
   
   ## 维护技巧
   
   为了保持组织：
   
   1. **每周**：整理新的下载
   2. **每月**：审查和归档已完成的项目
   3. **每季度**：检查新的重复文件
   4. **每年**：归档旧文件
   
   ## 为您提供的快速命令
   
   ```bash
   # 查找本周修改的文件
   find . -type f -mtime -7
   
   # 按类型整理下载
   [自定义设置命令]
   
   # 查找重复文件
   [自定义命令]
   ```
   
   想要整理另一个文件夹吗？
   ```

## 高级技巧

1. **从小处着手**：从混乱的文件夹（如下载）开始，建立信任
2. **定期维护**：每周对下载进行清理
3. **一致的命名**：使用 "YYYY-MM-DD - 描述" 格式为重要文件命名
4. **积极归档**：将旧项目移动到归档而不是删除
5. **保持活动分离**：在活动工作和归档工作之间保持清晰的界限
6. **相信过程**：让 Claude 处理物品放置的认知负担

## 最佳实践

### 文件夹命名
- 使用清晰、描述性的名称
- 避免空格（使用连字符或下划线）
- 尽可能具体："client-proposals" 而不是 "docs"
- 使用前缀进行排序："01-current"，"02-archive"

### 文件命名
- 包含日期："2024-10-17-meeting-notes.md"
- 描述性： "q3-financial-report.xlsx"
- 避免在名称中使用版本号（使用版本控制代替）
- 移除下载残留物："document-final-v2 (1).pdf" → "document.pdf"

### 归档时机
- 6个月以上未触及的项目
- 可能以后会参考的已完成工作
- 迁移到新系统后的旧版本
- 您犹豫是否删除的文件（先归档）
