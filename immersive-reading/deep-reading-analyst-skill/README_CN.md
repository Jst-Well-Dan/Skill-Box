## 这是什么？

**Deep Reading Analyst** 是一个为 Claude AI 设计的分析技能，它不是简单的文章摘要工具，而是一个**思维框架工具箱**。

### 核心能力

- 🔍 **结构提取** - 快速理解文章脉络和逻辑
- ✅ **完整性检查** - 发现信息缺口
- 🎯 **批判性分析** - 评估论证质量和逻辑漏洞
- ⚠️ **风险识别** - 发现潜在失败模式
- 🔗 **关系映射** - 理解系统性联系

### 适用场景

✅ 阅读商业策略文章
✅ 分析学术论文
✅ 评估技术方案
✅ 准备决策资料
✅ 学习新领域知识
✅ 准备团队讨论

## 为什么需要它？

### 问题：信息过载时代的阅读困境

- 📚 **内容太多**：每天面对海量文章，没时间深度思考
- 🤔 **理解浅表**：读完记不住，缺乏批判性思考
- 💭 **思考零散**：有感悟但不系统，难以应用
- ⏰ **效率低下**：重复阅读同类内容，没有积累

### 解决方案：系统化的分析框架

这个 skill 提供：

1. **结构化方法** - 5 个精选框架覆盖 95% 阅读场景
2. **渐进式深度** - 从 15 分钟快速理解到 120 分钟研究级分析
3. **可执行输出** - 不只是洞察，还有具体行动步骤
4. **适应性强** - 根据内容类型和目标自动调整

## 工作原理

### 5 步分析流程

```
1️⃣ 初始化
   ↓ 理解你的目标和需求

2️⃣ 结构理解
   ↓ 提取文章框架和核心概念

3️⃣ 应用思维框架
   ↓ 根据深度级别使用不同框架

4️⃣ 综合输出
   ↓ 生成符合你目标的结果

5️⃣ 知识激活
   ↓ 提供可执行的行动步骤
```

### 框架选择逻辑

Claude 会根据三个因素自动选择框架：

#### 1️⃣ 深度级别（你指定或默认）

```
Level 1 → SCQA + 5W2H
Level 2 → + Critical Thinking + Inversion
Level 3 → + Systems Thinking
```

#### 2️⃣ 内容类型（自动识别）

```
商业文章 → 强调 SCQA + Inversion + Systems
研究论文 → 强调 5W2H + Critical Thinking
技术指南 → 强调 SCQA + 5W2H
观点文章 → 强调 Critical Thinking + Inversion
```

#### 3️⃣ 用户目标（从你的描述推断）

```
解决问题 → 优先 Inversion + Critical Thinking
学习理解 → 优先 SCQA + Systems Thinking
写作研究 → 优先 Critical Thinking + 5W2H
决策支持 → 优先 Inversion + Systems Thinking
```

### 渐进式披露设计

skill 采用三层加载策略，确保效率：

```
第 1 层：元数据（始终加载）
  ↓ skill 名称和描述 (~100 字)

第 2 层：SKILL.md（触发时加载）
  ↓ 核心工作流程 (~190 行)

第 3 层：References（按需加载）
  ↓ 详细框架文档（只在需要时加载）
```

**好处：**
- ✅ 减少初始上下文占用
- ✅ 加快响应速度
- ✅ 保持深度分析能力

## 思维框架详解

### 1. SCQA 框架 - 结构化思维

**来源：** 麦肯锡咨询方法论
**用途：** 快速提取文章的逻辑结构

```
S (Situation)  - 背景情境
C (Complication) - 冲突/问题
Q (Question)    - 核心问题
A (Answer)      - 解决方案
```

**示例：**
```
S: 许多公司采用远程办公
C: 但团队协作效率下降
Q: 如何在远程环境下保持协作？
A: 采用异步决策机制
```

**何时使用：** 几乎所有文章都应先用 SCQA 建立结构框架

---

### 2. 5W2H 分析 - 完整性检查

**来源：** 新闻写作和系统分析
**用途：** 发现信息缺口

```
What    - 是什么？
Why     - 为什么？
Who     - 谁相关？
When    - 什么时候？
Where   - 在哪里？
How     - 怎么做？
How much - 成本/规模如何？
```

**示例输出：**
```
✅ 充分说明：What, Why, How
⚠️ 部分说明：Who, When
❌ 缺失：Where, How much

关键缺口：文章未说明实施成本和时间周期
```

**何时使用：** 评估信息完整性，特别是技术文档和研究论文

---

### 3. Critical Thinking - 批判性思维

**来源：** 逻辑学和论证理论
**用途：** 评估论证质量

**检查维度：**
- 📊 证据质量（样本量、来源可靠性）
- 🔗 逻辑推理（是否存在谬误）
- ⚖️ 平衡性（是否考虑反面观点）
- 🎯 结论支撑度（证据能否支持结论）

**常见逻辑谬误识别：**
```
❌ 草率概括 - 样本太小就下结论
❌ 虚假因果 - 误把相关当因果
❌ 稻草人谬误 - 歪曲对方观点再批驳
❌ 滑坡谬论 - 夸大后果
```

**示例输出：**
```
论证强度：6/10

优点：
- 引用了 3 项研究支持
- 承认了方法的局限性

缺点：
- 样本量偏小（n=50）
- 忽略了反面案例
- 存在确认偏误
```

**何时使用：** 分析观点文章、评估决策依据、学术论文审查

---

### 4. Inversion Thinking - 逆向思维

**来源：** 查理·芒格的投资哲学
**用途：** 识别风险和失败模式

**核心问题：**
```
❓ 如何确保这件事失败？
❓ 哪些假设如果错了会很危险？
❓ 文章没提但可能出问题的地方？
```

**示例输出：**
```
🚨 失败模式分析：

1. 依赖单一关键人物
   → 风险：此人离职导致项目中断
   → 缓解：建立知识文档和备份方案

2. 假设用户行为不变
   → 风险：市场变化导致方案失效
   → 缓解：设计灵活调整机制
```

**何时使用：** 决策前风险评估、战略规划、方案审查

---

### 5. Systems Thinking - 系统思维

**来源：** Donella Meadows 系统动力学
**用途：** 理解要素间的关系和反馈循环

**核心概念：**
- 🔄 **反馈循环** - 增强循环 vs 平衡循环
- ⚡ **杠杆点** - 四两拨千斤的干预点
- 🕰️ **时间延迟** - 因果之间的滞后

**示例输出：**
```
系统关系图：

用户增长 ──➕增强──> 网络效应
     ↑                    ↓
     |                    ➕
     |                    ↓
  ⬅──➕─────────── 用户价值提升

反馈类型：增强循环（正向飞轮）
杠杆点：优化新用户邀请机制
时间延迟：效果需要 2-3 个月显现
```

**何时使用：** 分析复杂系统、理解长期影响、找干预点

## 技术架构

### 设计理念

**1. 渐进式披露（Progressive Disclosure）**

```
元数据层 → SKILL.md → References
  轻      →    中    →    重
```

- 减少初始加载
- 保持深度能力
- 按需加载细节

**2. 模块化框架**

每个框架独立：
- 可以单独使用
- 可以组合使用
- 易于理解和维护

**3. 适应性设计**

根据三个维度动态调整：
- 深度级别（用户指定）
- 内容类型（自动识别）
- 用户目标（对话推断）
